# Documentation Rules

## Khi nào phải cập nhật docs

- Thêm module mới.
- Đổi auth/role/security.
- Đổi API response.
- Đổi database schema.
- Đổi deploy/env variables.
- Đổi business flow: giao bài, nộp bài, chấm điểm, reminder.

## Cách viết docs

- Ghi mục tiêu trước.
- Có flow dạng step hoặc diagram text.
- Có bảng permission nếu liên quan role.
- Có ví dụ request/response nếu liên quan API.
- Có checklist test nếu là module quan trọng.
